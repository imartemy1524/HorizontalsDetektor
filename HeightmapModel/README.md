# HeightmapModel

U-Net with pretrained EfficientNet-B4 encoder for predicting dense heightmaps from contour map images.

## Overview

This model takes a photo of a topographic contour map as input and outputs a 2D heightmap — a per-pixel height prediction. It is trained on synthetic data generated by the `DatasetGenerator` project in this repo, with heavy augmentations to bridge the domain gap between clean rendered contour maps and real phone camera photos.

### Architecture

- **Encoder:** EfficientNet-B4 (pretrained on ImageNet) via `timm`
- **Decoder:** 5-stage U-Net decoder with skip connections and squeeze-and-excitation attention
- **Input:** `(B, 3, 512, 512)` RGB image
- **Output:** `(B, 1, 512, 512)` heightmap in `[0, 1]`
- **Parameters:** ~25M
- **Loss:** L1 + gradient (slope) loss + SSIM structural loss

### Augmentation Pipeline

Two-tier augmentation to simulate phone camera photos:

| Tier | Target | Transforms |
|------|--------|------------|
| **Spatial** | image + heightmap | random crop/resize, rotation, perspective warp, affine, elastic deformation, flips |
| **Contour color** | image only | color jitter, grayscale conversion, RGB shift, invert |
| **Occlusion** | image only | coarse dropout (simulates finger/pen/ruler) |
| **Photometric** | image only | brightness/contrast, hue/saturation, tone curve, gamma, shadows, noise (Gaussian/ISO/multiplicative), blur (defocus/motion/Gaussian/zoom), JPEG compression, sharpen, posterize |

cd HeightmapModel

### Requirements

- Python 3.10+
- PyTorch 2.0+
- CUDA GPU recommended (training on CPU is impractical)

## Usage

### 1. Generate training data

First, generate the dataset using the C# `DatasetGenerator`:

```
cd ../DatasetGenerator
dotnet run
```

This produces a directory with paired files:
- `{i}.jpg` — contour line image (512×512)
- `{i}.data` — raw heightmap (512×512 float32, row-major)
- `{i}.metadata.json` — generation parameters

> **Note:** Edit `Program.cs` to set `outputDir` to your desired path before running.

### 2. Preview dataset + augmentations

Verify that data loading and augmentations look correct:

```
python dataset.py /path/to/dataset --num-samples 4
```

This saves a `dataset_preview.png` showing augmented contour images alongside their ground truth heightmaps.

Pass `--no-augment` to see the raw unaugmented data.

### 3. Train

Basic training:

```
python train.py --data-dir /path/to/dataset --epochs 100
```

Full example with all options:

```
python train.py \
    --data-dir /path/to/dataset \
    --epochs 100 \
    --batch-size 8 \
    --lr 1e-3 \
    --weight-decay 1e-4 \
    --warmup-epochs 3 \
    --freeze-encoder-epochs 2 \
    --grad-clip 1.0 \
    --ema-decay 0.999 \
    --l1-weight 1.0 \
    --gradient-weight 0.5 \
    --structure-weight 0.2 \
    --patience 20 \
    --run-name my_experiment \
    --save-every 10 \
    --vis-every 5 \
    --num-workers 4
```

#### Key training flags

| Flag | Default | Description |
|------|---------|-------------|
| `--data-dir` | *required* | Path to generated dataset |
| `--epochs` | 100 | Number of training epochs |
| `--batch-size` | 8 | Batch size (reduce if OOM) |
| `--lr` | 1e-3 | Peak learning rate (decoder). Encoder uses `lr × 0.1` |
| `--freeze-encoder-epochs` | 2 | Freeze pretrained encoder for N initial epochs |
| `--warmup-epochs` | 3 | Linear LR warmup duration |
| `--patience` | 20 | Early stopping patience (0 to disable) |
| `--no-amp` | — | Disable mixed precision (if GPU issues) |
| `--resume` | — | Path to checkpoint to resume training from |
| `--max-samples` | — | Limit dataset size (for debugging) |

#### Monitoring

Training logs to TensorBoard:

```
tensorboard --logdir runs/<run_name>/tb_logs
```

Logged metrics:
- Train/val total loss, L1 loss, gradient loss, structure loss
- Learning rate (encoder and decoder)
- Prediction visualizations (input → prediction → ground truth)

#### Checkpoints

Each run saves to `runs/<run_name>/`:

| File | Description |
|------|-------------|
| `best.pt` | Best validation loss (full checkpoint with optimizer state) |
| `best_ema.pt` | Best model with EMA weights (for inference) |
| `latest.pt` | Most recent epoch (for resuming) |
| `epoch_NNNN.pt` | Periodic checkpoints |
| `args.txt` | Training configuration |
| `tb_logs/` | TensorBoard logs |

### 4. Resume training

```
python train.py --data-dir /path/to/dataset --resume runs/my_experiment/latest.pt
```

### 5. Inference

Run predictions on new images (phone photos of contour maps):

```
# Single image
python predict.py \
    --checkpoint runs/my_experiment/best_ema.pt \
    --input photo.jpg \
    --output result.png

# Directory of images
python predict.py \
    --checkpoint runs/my_experiment/best_ema.pt \
    --input photos/ \
    --output results/

# All output formats
python predict.py \
    --checkpoint runs/my_experiment/best_ema.pt \
    --input photo.jpg \
    --output results/ \
    --save-raw \
    --save-grayscale

# Test-time augmentation (slower, potentially better)
python predict.py \
    --checkpoint runs/my_experiment/best_ema.pt \
    --input photo.jpg \
    --output result.png \
    --tta

# Resize output to match input image dimensions
python predict.py \
    --checkpoint runs/my_experiment/best_ema.pt \
    --input photo.jpg \
    --output result.png \
    --original-size
```

#### Output formats

| Flag | Format | Description |
|------|--------|-------------|
| `--save-vis` (default) | `.png` | Colorized terrain visualization (side-by-side with input) |
| `--save-raw` | `.npy` | Raw float32 numpy array, loadable with `np.load()` |
| `--save-grayscale` | `.png` | 16-bit grayscale PNG (65535 = max height) |

## Tips

- **Batch size:** With AMP on a 12GB GPU, batch size 8 should fit. Reduce to 4 if you hit OOM.
- **Data volume:** More training data = better generalization. 10k samples is a reasonable start; 50k+ is better.
- **Encoder freezing:** The first 2 epochs freeze the pretrained encoder so the decoder learns meaningful features before fine-tuning starts. Increase this if you see early instability.
- **Gradient loss weight:** If contour lines look correct but heights between them are wrong, increase `--gradient-weight`.
- **Structure loss weight:** If large-scale terrain shape is wrong, increase `--structure-weight`.
- **Test-time augmentation:** `--tta` averages predictions over 4 flips. ~4× slower but can improve accuracy, especially on noisy inputs.
- **Domain gap:** If the model works on synthetic data but fails on phone photos, the augmentation pipeline may need tuning. Consider collecting a small set of real phone photos for validation.
